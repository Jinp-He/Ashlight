# Ashlight（灰烬之光）- 游戏设计文档

## 概述

**游戏名称**：Ashlight（灰烬之光）  
**游戏类型**：时间轴卡牌战斗游戏  
**引擎**：Unity  
**当前阶段**：核心系统开发

---

## 核心玩法

### 战斗系统

Ashlight 采用**基于时间轴的确定性战斗系统**，核心特点：

- **15格时间轴**：每个战斗单位拥有独立的时间轴
- **NvN架构**：支持多个玩家角色 vs 多个敌人的战斗
- **卡牌驱动**：玩家通过打出卡牌来影响战局
- **行动预测**：打出卡牌前可以预览战斗结果

### 时间轴机制

每张卡牌在时间轴上占据若干格子，分为三个阶段：

| 阶段 | 英文 | 描述 |
|------|------|------|
| 前摇 | Startup | 准备阶段，此时可被打断 |
| 生效 | Active | 效果触发阶段 |
| 僵直 | Recoil | 恢复阶段，无法行动 |

### 优先级系统

同一时间格内多个行动按优先级排序执行：

| 优先级 | 类型 | 数值 |
|--------|------|------|
| 最高 | 防御/拦截 | 100 |
| 高 | 迅捷 | 90 |
| 中高 | 攻击 | 80 |
| 中 | 治疗 | 70 |
| 中低 | 时间推迟 | 60 |
| 低 | Buff | 50 |

## 卡牌系统

### 卡牌效果类型

- **攻击（Attack）**：对敌人造成伤害
- **防御（Defense）**：获得护甲，减少受到的伤害
- **治疗（Heal）**：恢复己方单位血量
- **时间推迟（TimeShift）**：推迟敌人行动或加速己方行动
- **Buff**：施加增益/减益效果

### 伤害计算

```
实际伤害 = Max(0, 攻击伤害 - 护甲值)
剩余护甲 = Max(0, 护甲值 - 攻击伤害)
```

## 角色与敌人

### 玩家角色

- 每个角色拥有独特的卡组
- 角色有生命值、护甲等属性
- 可携带Buff状态

### 敌人

- 每种敌人有预设的行动模式
- 敌人的行动在时间轴上可见，玩家可进行策略应对
- 支持多敌人同时战斗

## 技术架构

项目采用**四层架构**设计：

```
Assets/Scripts/
├── Config/      📊 数据层 - 配置数据访问（Luban）
├── State/       💾 状态层 - 运行时状态 & 存档
├── Systems/     ⚙️ 系统层 - 游戏逻辑驱动
├── View/        🖼️ 表现层 - UI & 视觉
├── Common/      🔧 通用工具
└── Battle/      ⚔️ 战斗核心系统
```

### 核心设计原则

- **数据驱动**：所有数值来自配置表
- **POCO架构**：核心数据类不依赖MonoBehaviour
- **确定性计算**：相同输入必然产生相同输出
- **深拷贝支持**：用于战斗预测系统
