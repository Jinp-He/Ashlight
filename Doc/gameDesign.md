# Ashlight 游戏设计文档

> 本文档综合代码、配置表与设计文档，描述当前游戏的核心玩法与系统设计。

---

## 一、游戏概览

**Ashlight** 是一款以**卡牌驱动战斗**为核心的回合制策略 RPG。玩家扮演角色与敌人在**双轨时间系统**中交锋，通过出牌在即时收益与延迟收益之间做取舍，同时管理能量与过载等资源，形成具有策略深度的战斗循环。

- **核心体验**：资源管理 × 节奏控制 × 预判博弈
- **战斗驱动**：所有数值与行为来自 Luban 数据表（`CardInfo`、`CharaterInfo`、`EnemyInfo`、`EnemySkillInfo`）

---

## 二、核心战斗循环

战斗由**规划轨（Intent Track）**与**执行轨（Execution Track）**两条独立时间轨道组成。

```
                ┌──────────────────────────────────────┐
                │          规划轨 Intent Track           │
                │  所有单位随时间前进 ──────────► 到达终点  │
                └─────────────────────┬────────────────┘
                                      │
                         ┌────────────┴────────────┐
                         │                         │
                    【敌人到达】                【玩家到达】
                    生成行动意图               抽牌 → 出牌阶段
                         │                    ┌────┴────┐
                         │              迅捷行动      执行行动
                         │              立即生效      （最多1张）
                         │              不入执行轨       │
                         ▼                              ▼
                ┌──────────────────────────────────────┐
                │          执行轨 Execution Track        │
                │  已确定的动作随时间前进 ──────► 到达终点  │
                │                              动作生效  │
                └─────────────────────┬────────────────┘
                                      │
                                      ▼
                             单位回到规划轨起点
                             开始下一轮循环

※ 未打出执行牌时，单位跳过执行轨，直接回到规划轨起点
```

### 循环说明

| 情况 | 路径 |
|------|------|
| 打出执行牌 | 规划轨终点 → 出牌 → 执行轨 → 效果结算 → 规划轨起点 |
| 只打迅捷牌/跳过 | 规划轨终点 → 出牌 → **直接回规划轨起点**（跳过执行轨） |
| 敌人 | 规划轨终点 → 生成意图 → 进入执行轨 → 技能生效 → 规划轨起点 |

> **一句话理解**：规划轨决定"将要做什么"，执行轨决定"何时真正发生"。

## 三、卡牌系统

### 3.1 卡牌类型

| 类型 | 枚举值 | 生效时机 | 是否进入执行轨 | 每回合数量 |
|------|--------|---------|---------------|-----------|
| **迅捷行动**（Swift） | 0 | 立即生效 | 否 | 受能量限制，可打多张 |
| **执行行动**（Execution） | 1 | 执行轨到达终点时 | 是 | 最多 1 张 |

### 3.2 卡牌数据字段

| 字段 | 说明 |
|------|------|
| `Id` / `Name` | 唯一标识与显示名 |
| `CardType` | `Swift` / `Execution` |
| `Energy` | 出牌能量消耗 |
| `ExecutingCost` | 执行轨上的推进代价（决定延迟长短） |
| `Effects` | 效果列表（多态，见下节） |
| `BelongTo` | 所属角色（`CharacterEnum`） |
| `TargetType` | 目标类型（单体/全体/自身等） |
| `Rarity` | 稀有度 |
| `Description` | 卡牌描述文本 |

### 3.3 效果类型（Effect 多态）

| Effect 类型 | 用途 |
|-------------|------|
| `AttackEffect` | 造成伤害（可 AOE） |
| `AttackExtraEffect` | 条件增伤（目标携带特定状态时） |
| `AttackConditionalEffect` | 条件攻击 |
| `DefenseEffect` | 获得护甲 |
| `InterceptEffect` | 拦截/盾类效果 |
| `HealEffect` | 恢复生命 |
| `ClearRecoilEffect` | 清除僵直（Recoil）状态 |
| `PushCollisionEffect` | 推动目标行动条位置 |
| `TimeShiftAllEffect` | 全体行动条位移 |
| `CollisionEffect` | 碰撞结果处理 |
| `SwiftEffect` | 迅捷标记类效果 |
| `ChannelEffect` | 施加引导（Channeling）状态 |
| `TauntEffect` | 嘲讽 |
| `BuffEffect` | 通用 Buff |

### 3.4 玩家出牌阶段决策

每次出牌阶段，玩家面临三层核心决策：

1. **能量分配**：有限能量如何在多张迅捷牌之间分配
2. **是否出执行牌**：放弃执行牌可更快回到规划轨（加速循环），但损失延迟收益
3. **执行牌选择**：唯一执行牌名额给哪张（高伤害？强控制？防御准备？）

> **核心策略张力**：即时安全 vs 延迟收益 vs 循环节奏

### 4.2 属性说明

| `Speed` | 在规划轨/行动条上的推进速度 |
| `Draw` | 每回合抽牌数（均为 4） |

## 五、敌人系统

### 5.1 敌人数据（EnemyInfo）

| 字段 | 说明 |
|------|------|
| `Id` / `Name` | 标识与名称 |
| `Hp` | 生命值 |
| `Speed` | 规划轨推进速度 |
| `IntentionSet` | 意图集合（多组技能与时机） |

> 示例：`Enemy001` 小兵，Hp 50，Speed 5

### 5.2 敌人技能（EnemySkillInfo）

| 字段 | 说明 |
|------|------|
| `Id` / `Name` | 标识与名称 |
| `Effects` | 效果列表（与卡牌同源多态） |
| `ExecutingCost` | 执行轨推进代价 |
| `TargetType` | 目标类型 |

### 5.3 敌人 AI 行为

- 规划轨到达终点 → 从 `IntentionSet` **随机**选取一个技能意图
- 目标选取：当前选择**第一个存活的玩家**
- 技能进入执行轨后，到达终点时触发效果
- 敌人基础能量：2；不参与抽牌系统（`BaseDrawCount = 0`）

## 六、行动条系统（ATB）

### 6.1 离散段位行动条

| 参数 | 值 |
|------|-----|
| `MaxSegment` | 100 |
| `DefaultBaseRestartSegment` | 0 |

- 每轮 `Advance(Speed)` 使单位在行动条上推进
- 当 `CurrentSegment >= MaxSegment`，该单位触发行动
- 同帧多单位到达终点：**Speed 更高者优先**；Speed 相同时**玩家优先**

### 6.2 行动条区段（ActionBarZone）

| 区段 | 段位范围 | 说明 |
|------|----------|------|
| 起步 | 0–25% | 单位刚回到起点 |
| 正常 | 25–75% | 普通推进阶段 |
| 临界 | 75–99% | 即将触发行动 |
| 行动中 | ≥100% | 触发行动 |

### 6.3 UI 双轨行动条

行动条 UI 直观呈现规划轨和执行轨两条轨道：

- 图标在规划轨上匀速向右推进（`advanceSpeed` 默认 180 px/s）
- 打出执行牌后，图标转移到执行轨，从 `startX = -segmentWidth × executingCost` 开始推进
- 执行轨推进到终点 → 效果结算 → 图标回到规划轨起点

---

## 七、能量系统

- 每次**玩家回合开始**时，`CurrentEnergy` 重置为 `BaseEnergy`（来自 `CharaterInfo`）
- 每打一张牌消耗该牌的 `Energy` 值（多数卡牌为 1 点）
- 能量不足时无法打牌
- 回合结束时能量清零（不保留到下回合）

---

## 八、过载系统

过载是一种**惩罚性资源**，在激进出牌或特定效果下触发，影响下一轮的行动条起点。

### 8.1 过载等级与负债

| 过载等级 | 行动条后退（OverloadDebt） |
|----------|--------------------------|
| 轻度（Light） | +5 段 |
| 中度（Medium） | +12 段 |
| 重度（Heavy） | +25 段 |

- 过载最高等级：3（到达后无法继续堆叠）
- 行动条重启时消耗 `OverloadDebt`，使单位起点后移（即"更慢行动"）

### 8.2 过载策略含义

> 过载让你现在更强，但下一轮更慢。何时过载、过载多少，是战斗中重要的风险决策。

## 九、状态（Buff）系统

### 9.1 已定义状态枚举

| Buff | 说明 |
|------|------|
| `Channeling`（引导） | 持续施法状态，可被 `AttackExtraEffect` 利用增伤 |
| `Recoil`（僵直） | 行动受阻状态，可由 `ClearRecoilEffect` 清除 |

### 9.2 Buff 运行时结构

| 字段 | 说明 |
|------|------|
| `BuffId` | 字符串标识 |
| `Value` | 数值（强度） |
| `RemainingDuration` | 剩余持续回合数 |
| `StackCount` | 叠层数量 |

- 每回合结束时 `UnitState.UpdateBuffs()` 递减所有 Buff 的持续时间
- 护甲在每回合结束时**清零**

## 十、数据表结构（Luban）

所有游戏数值均由 Luban 数据表驱动，对应关系如下：

| 表名 | Excel 源 | JSON 输出 | 说明 |

| `TbEncounter` | `Enemy/#Encounter.xlsx` | `enemy_tbencounter.json` | 关卡遭遇（敌人组合） |

## 十二、系统演进说明

游戏战斗系统经历了三代演进，当前代码处于过渡期：

| 维度 | Timeline 系统（旧） | ATB 系统（中） | 双轨系统（当前设计） |
|------|-------------------|--------------|-----------------|
| 驱动模型 | 离散时间格（15格）推进 | 离散行动条段位推进 | 双轨道独立推进 |
| 行动决定 | 卡牌插入时间格 | 回合内出牌 | 规划轨到达后出牌 |
| 生效时机 | 时间格到达时 | 回合内结算 | 执行轨终点（迅捷立即） |
| 核心张力 | 时间格位置博弈 | 能量/过载权衡 | 即时安全 vs 延迟收益 vs 循环节奏 |

> **技术债提示**：`TurnResolver` 中能量消耗仍为每卡固定 1 点（待接 `card.Energy`）；旧 `Timeline` 路径与新 `ActionBarResolver` 路径并存，部分 Effect 类型仅在旧路径中完整实现。
